This is Web App that utilizes Laravel/PostgreSQL as backend and ReactJs as frontend.

Introduction

Task-challange is a Web App that manages Customers and verify their phone numbers using a number valiation API.

Laravel is used to build the RESTful APIs and serves as the back-end. This solution contains two seperate laravel apps: CustomerAPI and NumverifyAPI
CustomerAPI micros-service contains APIs responsible for Add, Update, Delete and Get all.
CustomerAPI Github-repo
NumverifyAPI app conatins 1 micro-service for verifying the number provided by the customer.
NumverifyAPI Github-repo

PostgreSQL is used as the database management system.

Reactjs is used as the frontend library for demonstrating the API's functionalities.
The front-end also demonstrates the use of tailwind.css a CSS framework.
Reactjs Github-repo

All of the solution is packed into a Docker image.
Instructions will follow on how to configure and run the container.

It also demonstrates features of the numverifyAPI API.

-Parts that should be improved:
NumverifyAPI app should use a light-weight framework(Laravel's Lumen)
The front-end design and visuals should be improved.
-Parts that are efficient:
The app is containarized so it can be deployed on any environment.
The APIs utilize a workflow convention of validate, persist, return.

Instruction on running the docker container

Docker Depolyment

1- Instructions on configuring and running the docker container.
docker-compose.yml is found in task-challange. This file defines the multi-container Docker application, and is used to configure each app's services.
2- Sail is used for interacting with Laravel's default Docker development environment.
The Composer dependencies are installed with
composer require laravel/sail --dev
php artisan sail:install
This will create the docker-compose.yml and configure the first app CustomerAPI and postgresql.
Then add "- laravel.phone-api" under the depends on tag.
3- In the CustomerAPI .env file you need to add
APP_PORT=8000
PHONE_API_HOST=task-challange-laravel.phone-api-1
PHONE_API_PORT=80
and edit the following
DB_CONNECTION=pgsql
DB_HOST=task-challange-pgsql-1
DB_PORT=5432
DB_DATABASE=task_challange
DB_USERNAME=task_challange
DB_PASSWORD=task@123
4- Defining the NumverifyAPI app.
Note that we are not exposing the NumverifyAPI's port because the CustomerAPI is the only app communicating with it.
Add in the docker-composer.yml the following.
laravel.phone-api:
build:
context: ./vendor/laravel/sail/runtimes/8.1
dockerfile: Dockerfile
args:
WWWGROUP: '${WWWGROUP}'
image: sail-8.1/app
extra_hosts:
- 'host.docker.internal:host-gateway'
environment:
WWWUSER: '${WWWUSER}'
LARAVEL_SAIL: 1
XDEBUG_MODE: '${SAIL_XDEBUG_MODE:-off}'
XDEBUG_CONFIG: '${SAIL_XDEBUG_CONFIG:-client_host=host.docker.internal}'
volumes:
- "../NumberValidation:/var/www/html"
networks:
- sail
5-Defining the ReactJS app
We will create a Dockerfile in the react app directory and refernce it in the docker-compose.yml
Add in the docker-compose.yml the follwing.
test_front:
build:
context: ../react-website
dockerfile: Dockerfile
ports:
- "3000:3000"
container_name: test_front
networks:
- sail
depends_on:
- laravel.test
Create the Dockerfile in the react dir with the following.
FROM node:18.12.1
WORKDIR /usr/src/app/my-app
COPY . ./
RUN npm install
EXPOSE 3000
CMD ["npm", "start"]
